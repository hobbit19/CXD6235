Integrating Android Messages and Carrier Services

Android Messages and Carrier Services provide devices with SMS, MMS, and RCS functionality. To integrate these apps, follow the steps outlined below. Partners can also use MR updates to replace existing default messaging apps with Android Messages.

Enable Android Messages and RCS on devices

Complete each of the following steps to ensure that devices are correctly RCS-enabled:

  1. Preload and configure Android Messages.
  2. Preload and configure the Carrier Services APK.
  3. Set System Property flags for GTS tests.

Preload Android Messages

Android Messages, available on the Play Store, can be preloaded on an Android device.

  1. Download the latest APK from the Messages Drive.
  2. Copy the appropriate apk version, based upon screen size and chipset version, to the device /system/app folder.
  3. Set the app as the default SMS/messaging app.*
  4. Add the Android Messages app icon to the hot seat, the application dock at the bottom of the default home screen.*
  5. (Optional) Configure any required CarrierConfig as described in Android Messages Carrier Configuration. Contact your account manager for more information.
* Depending on your agreement with Google, these steps may be mandatory.

Optional: Enable AOSP cell broadcast (CBS)

AOSP provides a basic cell broadcast (CB) receiver app. This app allows users to configure some, but not all, ranges of CB channels. OEMs can modify the CB app if the AOSP version does not satisfy channel requirements.

If the cell broadcast receiver app is installed on the device, users can launch it from Android Messages by going to Settings > Advanced > Wireless Alerts. This menu will appear only if the CB app is installed. The app uses hidden SmsManager APIs, such as SmsManager.enableCellBroadcastRange().

Preload the Carrier Services APK

The Carrier Services APK is a headless APK that contains the RCS stack. The Carrier Services APK must be installed on Android devices in order to properly enable RCS.

  1. Download the latest APK from the Carrier Services Drive folder.
  2. Add the APK to the Android source directory where prebuilt apps should be, depending on the OEM's configuration. Update the makefile in that directory.
    • Make sure the APK will be installed in /system/priv-app by adding the following to the relevant Android.mk file:
      LOCAL_PRIVILEGED_MODULE := true
    • Make sure the APK's cert is preserved during the build process so that the Play Store update will work. This means that the build process should not re-sign the APK with other certs. Include the following in the relevant Android.mk file:
      LOCAL_CERTIFICATE := PRESIGNED
  3. Configure the OEM device build file to load the APK in the system image.
  4. Add the following to the device's sysconfig XML sources, etc/sysconfig, to disable battery optimization and data saver for the Carrier Services APK:
    <!-- If CarrierServices is installed, it must always have network access to reliably receive IMS messages. -->
    <allow-in-power-save package="com.google.android.ims" / >
    <allow-in-data-usage-save package="com.google.android.ims" / >
  5. For devices running Android 8.x and later, add the following to the vendor.xml file:
    <carrier_config>
      <string name="config_ims_package_override_string">com.google.android.ims</string>
    </carrier_config>
    If the carrier override flag is not set, the system will fall back to the service specified in the device or product overlay.
    If products within the same device family require a different IMS Service, directly specify mcc-mnc keys in the vendor.xml file. For example:
    <carrier_config_list>
      <carrier_config>
        <string name="config_ims_package_override_string">com.google.android.ims</string>
      </carrier_config>
      <carrier_config mcc="310" mnc="016">
        <string name="config_ims_package_override_string">com.oem.ims</string>
      </carrier_config>
    </carrier_config_list>
  6. For devices running Android 8.x and later, add the following to the /etc/permissions/privapp-permissions.xml file:
    <privapp-permissions package="com.google.android.ims">
      <permission name="android.permission.MODIFY_PHONE_STATE"/>
      <permission name="android.permission.READ_PRIVILEGED_PHONE_STATE"/>
    </privapp-permissions>

Set System Property flags for GTS tests

To ensure RCS apps are correctly installed and configured, new device builds should run GTS. If you are preloading Android Messages or Carrier Services, set the ro.com.google.acsa property as follows:

  1. Add the system property value ro.com.google.acsa to the system image by including the following line in the device.mk file:
    PRODUCT_PROPERTY_OVERRIDES += ro.com.google.acsa=true
  2. The property ro.product.first_api_level should be unset (removed) for the first build of a product, and set to the correct API level value for all subsequent builds. If the flag is unset, Android will assign Build.VERSION.SDK_INT to ro.product.first_api_level. For more information, see the CTS documentation on source.android.com.

MR (OTA) updates to Android Messages for in-market devices

Some OEMs or operators may require updates to in-market devices to ensure that they are RCS-enabled. This can be achieved by an OTA update to replace existing default SMS apps with Android Messages. The OTA can be pushed using a maintenance release (MR). Any device running Android 4.4 and later can be updated.

OTA update guidelines

Follow the guidelines below in order to make the user experience frictionless.

Trampoline apps

A trampoline app retains users' original app icon locations. For Android Messages, a trampoline app will replace the original OEM SMS app icon location. When users tap on the SMS app icon, the Android Messages app will open.

Trampoline apps must:

  • have the original OEM SMS app’s package name.
  • be signed with the same key as the original OEM SMS app.
  • have an activity matching the original OEM SMS app's launcher activity, except that the trampoline app activity will launch into Android Messages’ main activity.
  • ensure that users’ default carrier messaging app selection, along with the location of their original SMS app’s icon on the device, is retained.

Inform users

When updating the SMS app using an OTA update, we strongly recommending providing users with adequate notification and a choice of whether or not to proceed. We recommend including the following dialogue to notify users:

This update will upgrade your original SMS/carrier messaging app to Android Messages.
The new app’s icon will replace your original app’s icon location, and will appear as a circular blue icon surrounding a white conversation bubble.
The new app supports Rich Communications Services (RCS), an upgraded standard for SMS. New features include an upgraded user interface, higher-resolution image and video sharing, typing notifications, and open group messaging (on a growing number of supporting carriers).

Create a trampoline app

To create a trampoline app:

  1. Download the Android Messages trampoline app template code.
  2. Replace the package name in the trampoline app’s AndroidManifest.xml from package="com.android.mms" to your original SMS app’s package name.
  3. Change your overlay config file. This file can be found somewhere in your tree, for example overlay/frameworks/base/core/res/res/values/config.xml. Change the property of default SMS application to Android Messages:
    <string name="default_sms_application" translatable="false">com.google.android.apps.messaging</string>
  4. Compile the customized trampoline app APK.
  5. Install your customized trampoline app as an update to your original SMS app.
  6. Install Android Messages and CarrierServices.apk with appropriate permissions. See Preload Android Messages and Preload the Carrier Services APK for instructions.
  7. Ensure the correct behavior by verifying these results
    1. The Android Messages icon now appears in all locations the user previously had the original SMS app icon.
    2. When the SMS app icon is tapped, Android Messages launches.
    3. The default carrier messaging app selection prior to the update is retained after the update.

Other resources

For further questions or concerns about Android Messages or Carrier Services, email rcs-support@google.com.
